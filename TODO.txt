AHORA QUE YA TENEMOS LA IDEA DE LA INTERFAZ GRÁFICA
PODEMOS HACER EL TEMA DE LOS GESTOS

✳️ HAY QUE HACER UN CONJUNTO DE FUNCIONES QUE MANEJEN TODAS LAS POSIBLES OPCIONES 
  DE GESTOS. POR EJEMPLO: UNA FUNCIÓN PARA ABRIR UNA CARPETA DEL EXPLORADOR QUE SE PASA POR PARAMETRO
  O ABRIR UN PROGRAMA QUE SE PASA POR PARAMETRO. O SUBIR EL VOLUMEN, ETC. ESTAS FUNCIONES SERÁN UTILIZADAS POR
  LOS GESTOS PARA EJECUTAR SU ACCIÓN. PARA ELLO LA CLASE GESTO TIENE DOS PROPIEDADES EL NOMBRE PARA IDENTIFICARLO
  Y UNA FUNCIÓN QUE EJECUTA LO QUE HAGA EL GESTO. ESTA FUNCIÓN DEL GESTO DEBE EJECUTAR UNA DE ESTAS FUNCIONES GENERICAS
  POR EJEMPLO PODRÍA SER: open_explorer_at("C://programs/minecraft"), HABRÍA QUE IMPLEMENTAR ESA FUNCIÓN open_explorer_at
✳️ DESPUÉS DE HACER ESTE CONJUNTO DE FUNCIONES GENÉRICAS, HAY QUE DETECTAR EVENTUALMENTE CIERTOS GESTOS 
  QUE DE MOMEMNTO VAMOS A IMPONER NOSOTROS COMO PREDETERMINADOS.
✳️ EL RECONOCIMIENTO DE GESTOS ES AUTOMÁTICO, EL USUARIO NO TIENE QUE DARLE A NINGÚN BOTÓN DEL TECLADO O DE LA INTERFAZ
  PARA QUE EL PROGRAMA SEPA QUE INICIÓ O TERMINÓ DE HACER UN GESTO.
✳️ UNA VEZ QUE UN GESTO HAYA SIDO RECONOCIDO DEBEMOS ENVIARLE EL USUARIO FEEDBACK. CREO QUE LA MEJOR MANERA
  ES DESTACAR EL GESTO ACTIVADO DE LA BARRA LATERAL CAMBIÁNDOLE EL COLOR DE FONDO DE JUSTO ESE GESTO ACTIVADO 
  Y LÓGICAMENTE DESPUÉS EJECUTAR LA TEAREA ASOCIADA AL GESTO INCLUSO PODRÍAMOS HACER SONAR UN AUDIO CORTO TIPO
  NOTIFICACIÓN CUANDO EL PROGRAMA DECTECTA UN GESTO
✳️ EL PRÓXIMO PASO SERÁ CREAR LA VENTANA DE CONFIGURACIÓN DE GESTOS QUE SE ACCEDERÍA CON EL BOTÓN DEL ENGRANAJE
  DE LA BARRA LATERAL. DE MOMENTO, EN ESA VENTANA SE PODRÍA SOLAMENTE MODIFICAR EL NOMBRE O LO QUE HACEN ESTOS GESTOS
  DE TODAS LAS POSIBLES ACCIONES QUE PUEDE REALIZAR QUE LE VAYAMOS A OFRECER AL USUARIO.
